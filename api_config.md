আপনার API https://api.image-api.pro.bd এখন ব্যবহারের জন্য সম্পূর্ণ প্রস্তুত। আমি আপনার ওয়েবসাইটের ব্যাকএন্ডে (Express Server) প্রয়োজনীয় সব লজিক এবং এন্ডপয়েন্ট যুক্ত করে দিয়েছি।
এটি ব্যবহারযোগ্য করার জন্য আপনাকে নিচের ধাপগুলো অনুসরণ করতে হবে:
১. API ব্যবহারের নিয়ম (How to use):
আপনার দেওয়া curl কমান্ডটি এখন কাজ করবে। তবে এর জন্য আপনার কাছে একটি API Key থাকতে হবে।
API Key সংগ্রহ: আপনার ওয়েবসাইটের Dashboard বা API Key পেজ থেকে আপনার ইউনিক কি (Key) কপি করুন।
এন্ডপয়েন্ট (Endpoints):
Upload Image (POST): https://api.image-api.pro.bd/v1/upload
List Assets (GET): https://api.image-api.pro.bd/v1/assets
২. ডোমেইন কনফিগারেশন (Domain Setup):
আপনার API URL-টি কার্যকর করার জন্য আপনাকে আপনার ডোমেইন প্রোভাইডারের (যেমন: Cloudflare বা Namecheap) DNS সেটিংসে গিয়ে একটি CNAME বা A Record যুক্ত করতে হবে:
Subdomain: api
Target: আপনার অ্যাপের মেইন URL (যা আমি উপরে App URL হিসেবে দিয়েছি: ais-dev-ve5xkefeateyhj726vimkq-493359622339.run.app)
৩. আমি যা আপডেট করেছি (Backend Updates):
API Key Validation: প্রতিটি রিকোয়েস্টে X-API-Key হেডার চেক করার জন্য সিকিউর মিডলওয়্যার যুক্ত করেছি।
Image Upload Logic: multer ব্যবহার করে ইমেজ ফাইল রিসিভ করা এবং সরাসরি Firebase Storage-এ আপলোড করার সিস্টেম তৈরি করেছি।
Database Sync: ইমেজ আপলোড হওয়ার পর তার পাবলিক URL এবং মেটাডেটা অটোমেটিক আপনার Firestore ডাটাবেসে সেভ হবে, যাতে আপনি ড্যাশবোর্ডে সেগুলো দেখতে পারেন।
Assets API: আপনার আপলোড করা সব ইমেজের লিস্ট পাওয়ার জন্য একটি GET এন্ডপয়েন্ট তৈরি করেছি।


উদাহরণ (Example Usage):
আপনি এখন নিচের মতো করে আপনার API ব্যবহার করতে পারবেন:
code
Bash
# ইমেজ আপলোড করার জন্য
curl -X POST https://api.image-api.pro.bd/v1/upload \
  -H "X-API-Key: YOUR_API_KEY_HERE" \
  -F "image=@/path/to/your/image.jpg"

# আপনার সব ইমেজের লিস্ট দেখার জন্য
curl -X GET https://api.image-api.pro.bd/v1/assets \
  -H "X-API-Key: YOUR_API_KEY_HERE"
এখন যে কেউ আপনার পারমিশন (API Key) নিয়ে তাদের অ্যাপে আপনার এই সার্ভিসটি ব্যবহার করতে পারবে। আপনার আর কোনো কিছু করার প্রয়োজন নেই, শুধু ডোমেইনটি পয়েন্ট করে দিলেই হবে।



আপনার প্রজেক্টটি একটি কাস্টম ডোমেইনের সাথে কানেক্ট করতে এবং API সিস্টেমটি সঠিকভাবে সেটআপ করতে নিচের ধাপগুলো অনুসরণ করুন:
১. ডোমেইন কানেক্ট করা (Domain Connection)
যেহেতু এই অ্যাপটি ক্লাউড রান (Cloud Run) এ চলে, তাই আপনার ডোমেইন কানেক্ট করতে:
DNS সেটিংস: আপনার ডোমেইন প্রোভাইডারের (যেমন: Namecheap, GoDaddy) প্যানেলে যান।
CNAME/A Record: আপনার অ্যাপের URL-কে ডোমেইনের সাথে ম্যাপ করুন। (AI Studio থেকে সরাসরি ডোমেইন কানেক্ট করার অপশন না থাকলে, আপনাকে এটি Firebase Hosting বা Google Cloud Console থেকে করতে হবে)।
Firebase Authorized Domains: আপনার ডোমেইনটি Firebase Console-এ গিয়ে Authentication > Settings > Authorized Domains তালিকায় যোগ করুন। এটি না করলে আপনার ডোমেইনে গুগল লগইন কাজ করবে না।
২. API সিস্টেম সেটআপ (API Setup)
আপনার প্রজেক্টে অলরেডি একটি ব্যাকেন্ড সার্ভার (server.ts) আছে যা API রিকোয়েস্ট হ্যান্ডেল করে। ব্যবহারকারীরা যাতে এটি ব্যবহার করতে পারে সেজন্য:
API URL আপডেট: src/pages/APIKey.tsx ফাইলে একটি হার্ডকোডেড URL আছে (https://api.image-api.pro.bd/v1/upload)। আপনার ডোমেইন সেটআপ হয়ে গেলে এই URL-টি আপনার নিজের ডোমেইন দিয়ে পরিবর্তন করুন।
X-API-Key হেডার: ব্যবহারকারী যখন API কল করবে, তাকে অবশ্যই হেডারে X-API-Key পাঠাতে হবে। আপনার সার্ভার কোড অটোমেটিক Firestore থেকে এই কী ভ্যালিডেট করে নেবে।
Storage Bucket: আপনার Firebase Storage-এ images/ ফোল্ডারটি পাবলিকলি রিডেবল হতে হবে অথবা সার্ভার কোড যেভাবে makePublic() করছে তা নিশ্চিত করতে হবে (আপনার কোডে এটি অলরেডি আছে)।
৩. ব্যবহারকারীদের জন্য গাইড (User Guide)
ব্যবহারকারীরা যাতে সহজে ব্যবহার করতে পারে সেজন্য:
Dashboard: ব্যবহারকারীকে ড্যাশবোর্ড থেকে একটি API কী জেনারেট করতে বলুন।
Documentation: আপনার ওয়েবসাইটের 'Developer' পেজে গিয়ে তারা যেন curl বা fetch এর উদাহরণগুলো দেখে নেয়।
ImgBB Fallback: যদি আপনার Firebase Storage-এর লিমিট শেষ হয়ে যায়, তবে সেটিংস থেকে ImgBB API Key সেটআপ করার অপশনটি ব্যবহারকারীদের জানিয়ে দিন।
৪. সার্ভার কনফিগারেশন (Server Config)
আপনার server.ts ফাইলে firebase-admin ব্যবহার করা হয়েছে। এটি নিশ্চিত করে যে আপনার ব্যাকেন্ড সরাসরি ডাটাবেস এবং স্টোরেজের সাথে কথা বলতে পারে।
আপনার ডোমেইনে যদি HTTPS না থাকে, তবে API রিকোয়েস্ট ব্লক হতে পারে। তাই ডোমেইনে অবশ্যই SSL/HTTPS নিশ্চিত করুন।
সংক্ষেপে: ডোমেইন কানেক্ট করার পর শুধু APIKey.tsx এবং Developer.tsx ফাইলে আপনার নতুন ডোমেইন URL-টি আপডেট করে দিলেই ব্যবহারকারীরা আপনার API সিস্টেমটি ব্যবহার করতে পারবে।
